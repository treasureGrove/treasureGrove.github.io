# 二次元 3D 技术美术作品集网站方案文档  
## 项目代号：**NEON TOON ORBIT**

---

## 1. 项目定位

本作品集网站不是传统的个人主页，而是一个具有强烈视觉记忆点的 **实时 3D 二次元角色互动体验站**。

网站核心体验为：

> 用户进入一个电影感的二次元赛博空间，中央悬浮一名高质量 3D 二次元角色。镜头围绕角色缓慢运动，粒子、光环、全息 UI、Shader Breakdown 和项目入口围绕角色展开。用户通过鼠标、滚动和点击操控镜头与视觉状态，逐步进入 Shader、VFX、角色渲染、工具链和项目案例模块。

整体目标是让访问者第一眼就感受到：

- 你不是普通前端 / 普通美术；
- 你具备 **技术美术、实时渲染、二次元角色表现、Shader、VFX、交互视觉设计** 能力；
- 网站本身就是一件作品。

## 3. 视觉方向

### 3.1 总体风格

风格不是低幼卡通，也不是纯游戏 UI，而是：

> **高级电影感二次元 × 赛博技术美术实验室**

画面应该具有：

- 暗色大背景；
- 强轮廓光；
- 深景深；
- 微弱体积雾；
- 漂浮粒子；

- 全息式 UI；
- 角色的二次元描边与高光；
- 类似游戏预告片开场的镜头运动。

## 6. 角色渲染要求

角色渲染采用 **3D 实时渲染 + 二次元风格化 Shader**。

### 6.1 核心 Shader 特性

角色需要具备以下渲染效果：

```text
1. Toon Ramp Shadow
2. 外轮廓描边 Outline
3. Rim Light 边缘光
4. Hair Specular 头发高光
5. Face Shadow 脸部阴影控制
6. Eye Highlight 眼睛高光
7. Emission 发光部件
8. MatCap / Stylized Specular
9. Debug View 切换
10. Post Processing 加持
```

## 7. 电影质感设计

为了避免变成廉价 3D 展示页，必须加入电影化处理。

### 7.1 镜头语言

镜头必须慢、稳、有重量。

建议：

```text
默认镜头：
轻微低角度仰视角色，提升角色气场。

鼠标移动：
镜头做 5° - 10° 的缓慢偏移。

滚动：
镜头围绕角色做有节奏的 Orbit Movement。

点击模块：
镜头推进或穿过角色背后的能量环，进入下一个页面。
```

### 7.2 光影设计

采用高对比电影布光：

```text
主光：
弱冷色正面光，保证角色面部可读性。

轮廓光：
强蓝紫 Rim Light，从角色背后打出剪影。

辅光：
低强度樱花粉 / 紫色环境光。

发光体：
背后能量环、角色身上的发光纹路、全息 UI。
```

### 7.3 后处理

必须配置后处理，否则很难出电影感。

建议使用：

```text
Bloom 辉光
Depth of Field 景深
Vignette 暗角
Film Grain 胶片颗粒
Chromatic Aberration 轻微色散
Tone Mapping 色调映射
Motion Blur 轻微运动模糊
```

注意：后处理要克制，不要变成满屏糊光。

## 8. 粒子系统设计

粒子不是装饰，而是空间氛围和交互反馈的一部分。

### 8.1 粒子类型

```text
1. 空气尘埃粒子
缓慢漂浮，增加空间感。

2. 环绕轨道粒子
围绕角色背后的能量环缓慢旋转。

3. 鼠标扰动粒子
鼠标经过时粒子轻微偏移或聚集。

4. 点击爆发粒子
进入模块时粒子向中心汇聚，然后爆发。

5. 长拖尾光点
偶尔从画面边缘划过，增加电影感。
```

### 8.2 粒子行为

默认状态：

```text
粒子缓慢绕角色运动。
```

鼠标移动：

```text
粒子受到轻微吸引，产生空间扰动。
```

滚动页面：

```text
粒子运动速度增加，形成镜头推进感。
```

点击模块：

```text
粒子汇聚到被点击的模块或角色核心，触发转场。
```

## 9. 首页交互分镜

### Scene 01：启动

时长：0 - 2 秒

画面：

```text
黑屏
微弱噪点
扫描线扫过
出现文字：
INITIALIZING TOON RENDER CORE...
```

氛围：

```text
安静、神秘、像游戏启动。
```

---

### Scene 02：角色显现

时长：2 - 4 秒

画面：

```text
角色轮廓先出现。
背后能量环慢慢亮起。
粒子从四周向角色聚合。
角色眼睛出现微光。
```

重点：

```text
不要一开始就全亮，要从暗处显现。
```

---

### Scene 03：镜头环绕

时长：4 - 6 秒

画面：

```text
镜头围绕角色旋转约 20° - 30°。
角色身上的 Rim Light 被带出来。
全息 UI 面板逐个出现。
```

出现模块：

```text
SHADER LAB
VFX LAB
CHARACTER RENDER
TOOLS / PIPELINE
PROJECT ARCHIVE
CONTACT
```

---

### Scene 04：可交互状态

6 秒后进入自由交互。

用户可以：

```text
移动鼠标控制镜头轻微偏移；
悬停模块触发角色和粒子反馈；
滚动页面进入分镜式内容；
点击模块进入具体作品页。
```

## 10. 鼠标交互设计

### 10.1 鼠标视差

鼠标 X 轴：

```text
控制相机左右轻微绕角色旋转。
```

鼠标 Y 轴：

```text
控制相机轻微俯仰。
```

响应方式：

```text
必须使用缓动，不允许直接跟随。
```

建议参数：

```text
最大水平偏移：8°
最大垂直偏移：5°
缓动系数：0.08 - 0.12
```

---

### 10.2 模块悬停反馈

当用户悬停某个模块时，应该有一组组合反馈。

例如悬停 `SHADER LAB`：

```text
1. SHADER LAB 面板发光；
2. 角色身上出现短暂扫描线；
3. 背后能量环加速旋转；
4. 角色表面出现 Debug 分层闪现；
5. 旁边浮现关键字：
   TOON RAMP / OUTLINE / RIM LIGHT / FACE SHADOW
```

悬停 `VFX LAB`：

```text
1. 面板周围出现火花粒子；
2. 地面出现魔法阵；
3. 屏幕出现轻微 Bloom 脉冲；
4. 角色手部出现能量聚集；
5. 浮现关键字：
   PARTICLES / DISSOLVE / SLASH / IMPACT
```

悬停 `TOOLS / PIPELINE`：

```text
1. 背景出现节点图；
2. 角色周围出现代码流；
3. UI 变成 Debug 风格；
4. 浮现关键字：
   PIPELINE / AUTOMATION / EXPORT / OPTIMIZATION
```

## 15. 动画风格

整体动画节奏要遵守：

```text

短爆发

```

不要所有东西一直高速运动。

```text

能量环缓慢旋转
```

建议：

```text

```

## 20. 性能要求

因为网站有大量 3D 和后处理，必须从一开始考虑性能。

### 20.1 模型优化

```text
角色模型面数控制
使用 LOD
减少透明材质
合并 Mesh
压缩 glTF
使用 Draco / Meshopt
贴图使用 KTX2 / WebP
```

### 20.2 粒子优化

```text
使用 GPU 粒子
尽量使用 Instancing
避免大量独立 Mesh
粒子数量根据设备动态调整
```

### 20.3 后处理优化

```text
移动端降低 Bloom 采样
低性能设备关闭 DOF
动态分辨率
限制 Pixel Ratio
```

### 20.4 设备分级

建议提供三档画质：

```text
High：
完整后处理、完整粒子、最高贴图

Medium：
减少粒子、降低 Bloom、关闭部分景深

Low：
保留角色和基础光效，关闭高级后处理
```

## 25. 最终目标效果

最终网站应该给人的感觉是：

> 打开网站像进入一个二次元游戏预告片。

> 角色的 Toon Shader、描边、Rim Light、粒子和后处理共同形成电影级画面。

> 整个作品集本身就是技术美术能力的展示。
